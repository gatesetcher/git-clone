### [ВВЕДЕНИЕ](#)
В рамках данной проектной практики главной целью была работа над интеграцией классического искусства и современных цифровых технологий. 

**Целью проектной практики** является разработка информационного веб-сайта для демонстрации коллекции исторических рам для картин, воссозданных или оцифрованных с помощью технологий трехмерного моделирования.

Для достижения поставленной цели были сформулированы и решены следующие **основные задачи**:

1.  **Изучение старинных картинных рам:** Проведение натурных исследований в музеях и на выставках для анализа стилистических особенностей, орнаментов и техник резьбы.
2.  **Изучение оборудования для 3D-сканирования и 3D-печати:** Технический анализ возможностей сканера *Shining 3D Einscan HX2* и подбор оптимальных материалов (таких как Wood PLA) для получения реалистичных копий.
3.  **Создание статического веб-сайта:** Разработка структуры и интерфейса платформы, которая будет служить онлайн-витриной для нашего каталога рам.
4.  **Освоение инструментов Git и Markdown:** Внедрение системы контроля версий для совместной работы группы и использование языка разметки для ведения проектной документации и наполнения контента сайта.

### 1. Общая информация о проекте
### Название проекта:
**«Проектирование и изготовление исторических рам с использованием 3D-сканирования»**

### Цели проекта и задачи проекта:
Основной целью проекта является создание технологического цикла по воссозданию объектов культурного наследия (исторических рам) и разработка информационного веб-сайта для их демонстрации. Проект направлен на сохранение классического искусства с помощью внедрения современных цифровых методов производства.

**1. Исследовательская деятельность:**
* Посещение музеев и выставок для детального изучения конструктивных особенностей и декора старинных рам.
* Анализ исторических стилей и выбор наиболее ценных образцов для оцифровки.

**2. Техническое проектирование и 3D-технологии:**
* Освоение работы с профессиональным 3D-сканером **Shining 3D Einscan HX2** для получения высокоточных цифровых моделей.
* Изучение технологии 3D-печати и проведение тестов материалов (сравнение стандартного **PLA** и композитного **Wood PLA**).
* Проектирование процесса постобработки напечатанных изделий для достижения визуального сходства с оригиналом.

**3. Информационные технологии:**
* Создание и верстка статического веб-сайта для публикации каталога оцифрованных рам.
* Использование системы контроля версий **Git** для организации совместной работы в группе из 18 человек.
* Документирование хода проектной деятельности с использованием языка разметки **Markdown**.

**4. Производственный этап:**
* Изготовление физических копий рам с использованием выбранных технологий.

### 2. Общая характеристика деятельности организации *(заказчика проекта)*

### Наименование заказчика:
**«Центр Искусств. Москва»**

### Организационная структура:
Организационная структура Центра представляет собой функциональную модель, типичную для крупных частных культурно-выставочных институций:
* **Руководство и кураторский отдел:** отвечает за концепцию выставок и подбор экспонатов из частных коллекций.
* **Экспозиционный отдел:** занимается дизайном выставочного пространства и техническим обеспечением выставок.
* **Отдел реставрации и консервации:** контролирует состояние предметов искусства и их обрамления.
* **Информационный отдел:** отвечает за наполнение сайта, работу с посетителями и образовательные программы.
* **Техническая служба:** обеспечивает функционирование мультимедийного оборудования и систем безопасности.

### Описание деятельности:
**«Центр Искусств. Москва»** — это уникальное арт-пространство на территории Храма Христа Спасителя, где экспонируются шедевры живописи из частных коллекций, ранее недоступные широкой публике. Деятельность организации включает:
* Проведение художественных выставок и тематических лекций.
* Сочетание классического искусства с современными мультимедийными технологиями.
* Экспертизу и атрибуцию предметов искусства.

### 3. Описание задания по проектной практике

Задание по проектной практике было разделено на два ключевых блока: обязательную часть, направленную на развитие IT-компетенций, и вариативную часть, посвященную глубокому погружению в предметную область проекта.

### 3.1. Обязательная часть
Основным техническим заданием в рамках практики являлось создание **сайта-презентации** с использованием технологий HTML/CSS, а также освоение инструментов документирования.

**В обязательную часть входило:**
1.  **Изучение синтаксиса Markdown:** Освоение языка разметки для ведения технической документации, подготовки отчетов и структурирования контента.
2.  **Ведение документации:** Регулярная фиксация хода работ и результатов в репозитории проекта с использованием Markdown-файлов.
3.  **Создание статического веб-сайта:** Разработка ресурса (не менее 5-ти страниц) со следующей структурой:
    *   **Домашняя страница:** Содержит краткую аннотацию проекта, его цели и актуальность;
    *   **Страница «О проекте»:** Развернутое описание технологического процесса;
    *   **Страница «Участники»:** Список всей группы с указанием личного вклада каждого студента в проект;
    *   **Страница «Журнал»:** Дневник проекта, включающий минимум три записи (поста) о прогрессе выполнения задач;
    *   **Страница «Ресурсы»:** Перечень полезных материалов, ссылок на техническую документацию оборудования и информацию о партнерах.

### 3.2. Вариативная часть
Вариативная часть была направлена на получение прикладных знаний в области исторического наследия и специфики работы с антиквариатом.

Для выполнения этой части и более глубокого изучения темы проектной деятельности, участники проекта неоднократно совершали **выезды в музеи и выставочные залы**. Данные визиты осуществлялись по специальному приглашению заказчика — **«Центр Искусств. Москва»**. 

В ходе этих выездов студенты:
*   Изучали подлинные образцы старинных рам в экспозиции Центра.
*   Консультировались с экспертами по вопросам аутентичности декоративных элементов.
*   Проводили тестовые замеры и анализ поверхностей для последующего 3D-сканирования.

### 4. Описание достигнутых результатов по проектной практике

В результате выполнения проектной практики были успешно реализованы все поставленные технические и исследовательские задачи. Основные итоги работы представлены ниже:

### 4.1. Разработка веб-ресурса
Итоговым продуктом стал полноценный **статический веб-сайт**, включающий пять обязательных разделов: «Домашняя страница», «О проекте», «Участники», «Журнал» и «Ресурсы».
*   **Стиль и интерфейс:** Дизайн сайта выполнен в **минималистичном стиле**, что позволяет сфокусировать внимание пользователя на визуальном контенте — фотографиях и 3D-моделях исторических рам.
*   **Технологический стек:** При верстке использовались чистые HTML и CSS, что обеспечило высокую скорость загрузки и адаптивность страниц.

### 4.2. Работа с системой контроля версий
В ходе практики была выстроена эффективная система командного взаимодействия:
*   **Репозиторий на GitHub:** Создан и настроен удаленный репозиторий, ставший единым источником правды для всех 18 участников группы.
*   **Культура коммитов:** Все изменения фиксировались регулярно. Каждый коммит сопровождался **осмысленным сообщением** (например, `feat: add navigation to resources page` или `docs: update project description in readme`), что позволило прозрачно отслеживать историю разработки.
*   **Git-процессы:** Участники освоили создание веток и слияние изменений (Merge Requests), что минимизировало конфликты в коде.

## 4.3. Документирование и Markdown
Был полностью изучен и внедрен синтаксис языка разметки **Markdown**. С его помощью:
*   Подготовлена проектная документация в репозитории.
*   Структурированы текстовые материалы для страниц сайта.
*   Оформлен данный отчет, что обеспечило единообразие и читаемость всей технической информации.
